线程睡眠
    1、睡眠状态会进入TimeWaiting状态
    2、可通过线程的interrupt()方法来中断睡眠
    3、睡眠结束不一定可以立即行动，需要等到cpu分配到时间片

yield
    1、调用yield()方法会将当前线程从Running状态转换成Runnable状态

优先级
    1、通过设置priority()方法设置优先级
    2、优先级高的线程会优先执行
    3、不过一切都有操作系统的调度器说的算，线程的优先级只是个参考

join
    1、join()方法会等待线程执行完毕
    2、join()方法会阻塞当前线程
    3、技术上是可以用sleep()方法来代替join()方法，但是不要知道目标线程什么时候处理完任务，所以join智能一些

打断线程
    1、stop方法已经被弃用了，因为容易丢失线程的数据
    2、interrupt()方法会打断线程，但是线程不一定会立即停止
    3、打断线程有两种情况，一种是非堵塞情况下被中断，一种是正在堵塞情况下被中断
    4、非堵塞情况下被中断，会把正常把中断标记置为true
    5、正在堵塞情况下被中断，会将中断标记置为false并抛出InterruptedException异常
    6、常见堵塞情况有sleep、wait、join
    7、所谓的两阶段终止其实就是自己处理被中断时的兜底处理
    8、LockSupport.park()相当于暂停线程，不消耗cpu资源，等待被中断而unpark()
